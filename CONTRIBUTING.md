# Contributing

## Content rules

- Keep `programme.md` as the main source of truth for the MiniRoos framework.
- Update audience pages to summarise or point back to the main programme page rather than duplicating large sections.
- Keep wording practical, public-facing and easy for families to scan.
- Use short headings and plain markdown.
- Put time-sensitive details in one place where possible to avoid drift.

## Current age-group wording

- Players born in `2020` are `U6`.
- Players born in `2019` are `U7`.
- Do not refer to `U5` as a separate WWJFC MiniRoos team unless the club structure changes.

## Editing guidance

- Prefer updating existing canonical pages instead of creating near-duplicate pages.
- Keep filenames short and predictable.
- Use local assets rather than hotlinking external images.
- Keep custom styling in `assets/main.scss`.

## Before finishing a change

- Check links still make sense.
- Check the content still reads clearly on mobile-sized screens.
- If possible, run a local Jekyll build before committing.
